Operating Environment

      This application is a desktop application which would run using Visual Studio 2019, the platform used to create it is Visual Basic (VB) .NET 4.75. The hardware which will be used to run this application is a desktop with 8GB of RAM, 1TB Hard Disk Drive (HDD) and a 7th Generation Intel i5 Processor; the operating system on the desktop would be Windows 10. The location of the desktop would be in the warehouse as the system is intended for that location. The servers would be local to the desktop so there would be no need to purchase server equipment for installation. The database is a local MySQL database which is used in Visual Studio during application development; it will be used to store the necessary data for the system and if the system needs to be improved it is simple to scale the database with the Visual Studio development environment.

Design and Implementation Constraints

      The system being developed is windows-based application, which will be developed using Visual Studio 2019. The application will use the Visual Basic .Net (VB.NET) architecture, along with a MySQL database for data storage. The VB.NET architecture is an object-oriented language which lends itself to simple and dynamic development of forms in the application. The MySQL database is a simple database tool to develop a functional database and when creating the connection from the forms to the database the SQL data client class would need to be imported in order to access the objects of the MySQL database.

Assumptions and Dependencies

Dependency

      The system will require the Microsoft .NET 4.7.2 in order to run the application.

Assumptions

      The Warehouse attendant will only have a single interaction with the system which will be to search for the location of a stock item.

      The Manager has the highest privilege and will have access to all forms in the system. The Manager will be able to view and search for employees in the warehouse, search for purchase orders, inventory items, deliveries. The manager can create new purchase orders, deliveries, and edit the list of inventory items.

      The Product clerk will have defined privileges; their access will be limited to purchase orders, deliveries and inventory. The product clerk can create new purchase orders and deliveries. The product clerk can search for purchase orders deliveries and inventory items.
