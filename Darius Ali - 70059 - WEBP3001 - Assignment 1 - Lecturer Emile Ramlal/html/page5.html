System Features

Login

Description

      The login screen will be the main control hub for granting access to key parts of the system once completed. The login screen will pull information from an Employee table in the database and compare the login credentials of the user to the credentials in the system. If the credentials are valid then the user is granted access based on their credentials. For example, if a manger logs in then the credentials would be validated, then access to the full system will be granted to the manager. In the case of a warehouse attendant logging in, their credentials would be validated and they would be granted access to search for an item in inventory.

Functional Requirements

  1. Must be able to validate user credentials.
  2. Must be able to direct to correct home screen when the users enters their login credentials.
  3. Must provide feedback to teh user in the event their credentials are invalid.
  4. Must provide feedback to user in the event that they did not enter a User ID.
  5. Must provide feedback in the event teh user did not enter a Password.
  6. Must provide feedback on successful login.

Non-Functional Requirements

  1. The text box for entering the user's password must utilize a mask to keep the password hidden.
  2. The user should not need to type their position but rather select it using a combo box.
